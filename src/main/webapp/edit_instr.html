Collatta Web Site Instructions

These instructions are for the colletta web site http://www.colletta.it.

Users

The site allows users  to login to get more permissions and to see more information. Currently there is no online mechanism for creating new
user accounts or passwords, they can only be updated by request to Mort Bay Consulting. Each user has
  • login name
  • full name
  • email address
  • preferred language
  • Zero or more roles
The following user roles are defined:

EDITOR
 -
 Once logged in, and EDITOR may select the edit mode and update the content of the pages.
OWNER
 -
 An owner of an apartment.  An owner may view bookings for their apartments and can approve or confirm those bookings. Owners may also set the manager  for bookings allocated to their apartments.
MANAGER
 -
 A manager may view full details of online bookings and can approve or confirm bookings for the apartments they are listed as managing.

Edit Instructions

Most pages on the website can be updated by a user with the EDITOR role.  Each web pages is a composite of text, properties and images. The text and
properties can be set for a particular language or for all languages. Once logged in, an EDITOR may enter edit mode by selecting the edit button in the bottom right corner all pages.  They can then use the site normally and the edit forms will appear at the bottom of every page.
 

Edit Page Text:

Most pages have some text displayed on them. This text can be entered in each of the supported languages in the EDIT PAGE TEXT boxes above. If text for a particular language is not available, then the English or Italian text will be displayed instead. The text entered may be formatted as follows:

!Heading - The text "Heading" formated as a section heading.
blank line - New paragraph
$$ - Line break
* my text - The text formated as a bullet list
# my text - The text formated as a numbered list
${name} - Replaced with the property called "name". See properties below.
{{{ html }}} - html markup
[text] - Bold text
[style:text] - Styled text. Known styles include: Bold, SubTitle, Label, Title
[[path:text]]
 -
 Link to this web site.
[[http://host/link:text]]
 - Link to another web site with text.
[[http://host/link:class:alt:image.jpg]]
 - Image link to another web site with text.
style:file.jpg - Styled image from current directory. Known styles include: Content,  BigThumb, AptSmall,  AptTiny,  Staff,
class:file.jpg - Inline image from current directory
class:alt:file.jpg - Inline image from current directory

Edit Page Properties:

Every page has a number of associated properties that are used to generate the page menus, indexes, titles and to expand the text properties entered above. Properties can be given a value that is common for all languages, or they may be given a value for each language. If there is no value for a language and no common value, then the english or italian value is used. Every page may have the following parameters:

name - The name of the page as it appears in the left side menu.
order - A value used to sort the order left menu.
title - The page title displayed at the top of the page.
blurb - A description of the page, normally used in the page above.
redirect - Display the same content as another page.

Each page may also have it's own set of special properties, these are often described in the "Page Instructions" at the bottom of the edit page. A page inherits all the properties from the page above in the website (eg. Home>Renting>Apartments inherits the properties from Home>Renting, which inherits properties from Home), so if you cannot find a property, look for it in the parent page.

Edit Page Photos:

Photos may be uploaded and deleted. Photos should be uploaded as jpg or gif files in the maximum size to be displayed. The names of the images should be such that they will sort alphabetically in the order they should be displayed down the page.

Bookings

Bookings for apartments may be taken from any user of the web site or entered by MANAGERs and OWNERs. 

Booking States

Requested - A booking is considered Requested if it is not Saturday to Saturday or no specific apartment has been identified.   Requested bookings do not hold an apartment. A manager must manually approve a requested booking.
Approved
 -
 A booking entered on the web will be automatically approved if it  is from saturday to saturday and for an available apartment.  When  approved, the apartment will be held for 48hours until the booking   is confirmed.    It can be confirmed manually by a manager OR by payment of a deposit.
Confirmed
 -
 Either a deposit has been paid or a Manager has manually confirmed the reservation. The booking will be held. Payment reminders will be sent 2 weeks before arrival
Expired
 -
 A deposit was not received within 48 hours of the approval.
Cancelled
 -
 A booking has been cancelled.

Payment

Payments may be made online via PayPal or entered manually by a Managers or Owners.   Managers and owners may also enter discounts for a booking.
PayPal allows for credit card transactions or for other forms of on-line transfers.  When money is received, the web site is automatically updated. Money can be
refunded upto 30 days after payment via paypay and will automatically be deducted for the booking.
